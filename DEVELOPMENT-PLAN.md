# 实时协作文档编辑器 - 开发计划 (DEVELOPMENT-PLAN.md)

本计划旨在指导“多人实时协作文档编辑器”的开发工作，详见 [AI编程马拉松题目.md](file:///d:/ai/AI%E7%BC%96%E7%A8%8B%E9%A9%AC%E6%8B%89%E6%9D%BE%E9%A2%98%E7%9B%AE.md)。本项目采用**多智能体 (Agent-A, B, C) 分支协作模式**。

---

## 1. 技术架构选型

### 前端 (Frontend)
- **框架**: React + TypeScript + Vite
- **富文本编辑器**: Quill + Yjs (CRDT)
- **UI**: Tailwind CSS + Lucide React

### 后端 (Backend)
- **实时通信**: y-websocket (Node.js)
- **数据库**: SQLite (持久化 Yjs binary updates)

---

## 2. 多智能体协作模型 (Multi-Agent Workflow)

### 分支管理策略 (Git Worktree)
- **`main` 分支**: 生产稳定分支。Agent-A 已完成基础框架搭建并合并至此。
- **并行开发阶段**: 剩余任务已重新拆分为 3 份，A、B、C 三个智能体将在各自的工作区内**同时并行**开发。
- **`Agent-A` 分支**: 负责前端深度协作 UI (光标、在线列表、格式工具栏)。
- **`Agent-B` 分支**: 负责后端持久化与版本历史逻辑 (SQLite 数据库、版本快照 API)。
- **`Agent-C` 分支**: 负责前端离线支持与状态恢复 (IndexedDB、断网重连、冲突提示)。

### 合并门禁 (Gatekeeping)
1. 智能体在各自的分支完成开发。
2. 运行该任务对应的自动化测试脚本。
3. 测试 100% 通过后，用户手动或由 AI 发起 `merge` 到 `main`。

---

## 3. 阶段性任务与测试验收标准 (Phases & Acceptance Tests)

### 第一、二阶段：基础设施与核心同步 (✅ 已完成)
- **完成者**: Agent-A (初始阶段)
- **已通过测试**: Test-A1, Test-B1, Test-A4

### 第三阶段：多智能体并行攻坚 (🔥 当前阶段)

#### 任务包 A (由 Agent-A 负责)
- **任务领域**: 前端协作 UI 与富文本体验
- **具体工作**: 
    - 实现其他用户的光标位置与颜色同步显示。
    - 实现侧边栏或顶部的在线用户列表动态更新。
    - 完善 Quill 编辑器的复杂格式支持 (加粗、斜体、列表、超链接)。
- **测试用例**: Test-B2 (光标), Test-B4 (用户通知), Test-A2/A3 (格式应用)。

#### 任务包 B (由 Agent-B 负责)
- **任务领域**: 后端持久化与版本历史
- **具体工作**: 
    - 集成 SQLite，实现文档的定时保存与增量快照。
    - 提供版本历史查询 API (获取历史列表、获取指定版本内容)。
    - 实现服务器端的版本回滚逻辑。
- **测试用例**: Test-C1/C2/C3/C4 (版本自动保存、内容正确性、回滚、上限)。

#### 任务包 C (由 Agent-C 负责)
- **任务领域**: 离线支持与数据一致性
- **具体工作**: 
    - 集成 `y-indexeddb` 实现前端浏览器本地持久化。
    - 处理断网期间的离线编辑逻辑，以及恢复网络后的自动同步。
    - 在特定场景下 (如版本回滚导致的冲突) 实现 Last-Write-Wins 策略。
    - 前端 UI 增加清晰的“离线/在线”状态提示。
- **测试用例**: Test-D1/D2/D3/D4 (离线编辑、冲突处理、长时恢复、状态提示)。

### 第四阶段：集成与性能测试 (所有智能体协同)
- **任务**: 各分支合并至 `main` 后，解决可能的集成冲突，进行最终 E2E 测试。
- **测试用例**: Test-E1 (大文档同步), Test-E2 (长时稳定), Test-E3 (50 用户并发), Test-B5 (三用户并发)。

---

## 4. 冲突解决策略 (CRDT + LWW)
- 实时编辑阶段：使用 Yjs 的 CRDT 自动处理。
- 离线恢复/回滚阶段：显式对比 `timestamp`，执行 **Last-Write-Wins (LWW)** 策略。
